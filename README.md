# Work Order System

アルマイト工場の現場業務を想定した  
**製造指示書（ラッキング工程）管理システム**です。

---

## ■ 開発背景

現場では現在、紙ベースで製造指示書を管理しています。

- 事務所が製図をコピー
- 製造番号・仕様・本数を手書き
- 枠ごとに管理
- 完了時間や最終チェックも紙記録

この運用では、

- 転記ミス
- 集計の手間
- 過去履歴の検索の難しさ

といった課題があります。

本システムはこれをWeb化し、  
**タブレット入力前提で現場負担を減らすこと**を目的としています。

## ■ 想定機能

- 枠単位での指示書作成
- 管理番号ごとの数量管理（例：5/8）
- ラッキング完了時刻入力
- 電解完了時刻入力
- 仕上げ場到着（完了）記録
- 管理番号による検索

---

## ■ 画面構成

本システムはタブレット入力を前提とし、  
画面数を最小構成としています。

- **Login**（持ち場アカウント）
- **G01：今日の指示書一覧**
  - ラッキング／仕上げタブ切替
- **G02：ラッキング完了登録**
  - 枠＋明細（複数行）をまとめて入力
- **G03：指示書詳細**
  - 電解完了・仕上げ場到着入力

※ 未着手のデータは一覧に表示せず、  
工程は「ラッキング完了」から開始する設計としています。

---

## ■ 状態管理の設計思想

本システムは `status` カラムを持たず、  
**nullable の時刻カラムのみで工程状態を管理**します。

- ラッキング完了  
  `racking_completed_at != null`

- 電解完了  
  `electrolysis_completed_at != null`

- 完了（仕上げ場到着）  
  `finishing_arrived_at != null`

完了判定は「仕上げ場到着」と定義しています。

工程状態を文字列で持たず、  
**実際の時刻データのみで状態を判断できる設計**としています。

## ■ 開発方針

実務現場を前提とした設計を重視しています。

- 入力負担を増やさない
- 紙運用との差を最小化
- 現場優先のワークフロー設計
- 状態は「データの事実（時刻）」で管理する
